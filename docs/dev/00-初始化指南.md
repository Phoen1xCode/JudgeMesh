## 阶段 2:Java 多模块父 POM(Day 1 下午 · A 主写)

最容易拖慢全队的一步,**晚一天交付,BCD 都没法启动**。

顶层 `pom.xml` 必备项:

1. `<packaging>pom</packaging>`,聚合 6 个 Java 子模块(`services/api`、`gateway`、`user-service`、`problem-service`、`submit-service`、`judge-dispatcher`)
2. `<dependencyManagement>` 锁版本:
   - `spring-boot-dependencies:3.5.0`(BOM 方式引入,而非 starter parent,便于自由继承)
   - `spring-cloud-dependencies:2025.0.0`
   - `spring-cloud-alibaba-dependencies:2025.0.0.0`
   - `seata-spring-boot-starter`(版本由 SCA BOM 管理,不要硬编码)
   - `mybatis-plus-spring-boot3-starter`、`lombok`、`mapstruct`、`micrometer-registry-prometheus`、`skywalking-apm-toolkit-trace`
3. `<build>` 统一插件版本:`maven-compiler-plugin`(Java 17)、`maven-surefire-plugin`、`jib-maven-plugin`(打镜像免 Dockerfile)
4. `<properties>` 统一 `project.build.sourceEncoding=UTF-8`、`maven.compiler.release=17`

`services/api/` 是**共享 DTO / Feign / MQ schema 模块**,所有业务服务都 depends on 它。**Day 2 之前必须把 `JudgeTask` / `JudgeResult` schema 落进去**——见 [12-五人分工 §Worker↔Dispatcher 协议](../design/12-五人分工.md#worker--dispatcher-协议a重点关注)。

**参考文档**:

- Spring Boot 3.5 文档:https://docs.spring.io/spring-boot/docs/3.5.x/reference/html/
- Spring Cloud 2025.0 文档:https://docs.spring.io/spring-cloud/docs/current/reference/html/
- Spring Cloud Alibaba:https://sca.aliyun.com/docs/
- SCA 版本对照表:https://sca.aliyun.com/docs/2023/overview/version-explain/
- Maven 多模块:https://maven.apache.org/guides/mini/guide-multiple-modules.html

## 阶段 5:CI/CD 骨架(Day 3 · E 主写)

`.github/workflows/` 下三条流水线足以覆盖前期:

| 文件                | 触发                                         | 做什么                                     |
| ------------------- | -------------------------------------------- | ------------------------------------------ |
| `ci-java.yml`       | PR + push 到 main(`paths: services/**`)      | `mvn -pl <changed> verify`                 |
| `ci-go.yml`         | PR + push(`paths: services/judge-worker/**`) | `go vet`、`go test ./...`、`golangci-lint` |
| `ci-frontend.yml`   | PR + push(`paths: frontend/**`)              | `npm ci`、`npm run lint`、`npm run build`  |
| `release.yml`(后置) | tag push                                     | 构建 + push Docker Hub + `kubectl apply`   |

镜像 tag 策略(对齐 [08-部署架构 §3.2](../design/08-部署架构.md)):

- main 分支:`latest` + `git-sha`
- 发布:语义化版本 `v0.1.0`

- GitHub Actions:https://docs.github.com/en/actions

- setup-java / setup-go / setup-node:https://github.com/actions
