## 1. 一键 / 半自动安装

### 1.1 macOS · Homebrew

```bash
brew tap homebrew/cask-versions
brew install --cask temurin@17
brew install maven go@1.22 node@20 docker kubectl helm git jq

# 把 java/go 设成默认
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 17)' >> ~/.zshrc
echo 'export PATH=$(brew --prefix go@1.22)/bin:$(brew --prefix node@20)/bin:$PATH' >> ~/.zshrc
source ~/.zshrc
```

> 多版本切换推荐 [SDKMAN](https://sdkman.io/)(Java)+ [nvm](https://github.com/nvm-sh/nvm)(Node)+ [g](https://github.com/stefanmaric/g)(Go)。

### 1.2 Linux(Ubuntu 22.04 / 24.04)

```bash
# JDK 17
sudo apt update && sudo apt install -y temurin-17-jdk maven git curl jq

# Go 1.22
curl -fsSL https://go.dev/dl/go1.22.10.linux-amd64.tar.gz | sudo tar -C /usr/local -xz
echo 'export PATH=/usr/local/go/bin:$PATH' >> ~/.bashrc

# Node 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Docker
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER && newgrp docker

# kubectl + helm
curl -LO "https://dl.k8s.io/release/v1.28.15/bin/linux/amd64/kubectl"
sudo install kubectl /usr/local/bin/
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
```

### 1.3 Windows

**强烈建议走 WSL2 + Ubuntu**,避免 isolate / cgroups 这类 Linux 原生特性兼容问题:

```powershell
wsl --install -d Ubuntu-24.04
```

进 WSL 后按 §1.2 操作。Docker Desktop 装在 Windows 侧,在 Settings → Resources → WSL Integration 勾选 Ubuntu。

## 4. 启动本地中间件

依赖 Docker。所有服务连本地中间件,**不依赖 K8s 集群**(集群是给 @sigufh 部署用的)。

```bash
cp infra/local/.env.example infra/local/.env
docker compose -f infra/local/docker-compose.yml --env-file infra/local/.env up -d
```

| 服务      | 本机端口     | 控制台 / 凭据                                                       |
| --------- | ------------ | ------------------------------------------------------------------- |
| MySQL     | 3306         | `judgemesh / judgemesh`,`users_db` / `problems_db` / `submits_db` 已建好 |
| Redis     | 6379         | 无密码                                                              |
| RabbitMQ  | 5672 / 15672 | http://localhost:15672 · `judgemesh / judgemesh`                   |
| MinIO     | 9000 / 9001  | http://localhost:9001 · `minioadmin / minioadmin`,3 桶已建好       |
| etcd      | 2379         | `curl http://localhost:2379/health`                                 |
| Nacos     | 8848 / 9848  | http://localhost:8848/nacos · 无鉴权                                |

冒烟:

```bash
docker compose -f infra/local/docker-compose.yml ps
mysql -h127.0.0.1 -ujudgemesh -pjudgemesh -e "SHOW DATABASES;"
redis-cli -h 127.0.0.1 ping
curl -s http://localhost:8848/nacos/actuator/health
```

> Nacos 首次启动需 30-60 秒,容器 healthy 之后再启业务服务。

## 5. Java 服务编译与运行

### 5.1 一次性把整个工程拉好依赖

仓库根目录:

```bash
mvn -T 4C -DskipTests clean install
```

这会把 SCA 2025.0.0.0 / SC 2025.0.0 / SB 3.5.0 全套 BOM 拉到本地 `~/.m2/repository`。**第一次跑要 5-15 分钟**,之后秒级。

> ⚠️ **如果 SCA 2025.0.0.0 在 Maven Central 解析不到**,把根 `pom.xml` 中 `<spring-cloud-alibaba.version>` 改为 `2025.1.0.0`(当前活跃版),其余无须改动。

### 5.2 跑单个服务

每个服务都能独立跑(前提:本地中间件已起):

```bash
mvn -pl services/user-service spring-boot:run
mvn -pl services/problem-service spring-boot:run
mvn -pl services/submit-service spring-boot:run
mvn -pl services/judge-dispatcher spring-boot:run
mvn -pl services/gateway spring-boot:run
```

冒烟:

```bash
curl -s localhost:8081/actuator/health   # user-service
curl -s localhost:8082/actuator/health   # problem-service
curl -s localhost:8083/actuator/health   # submit-service
curl -s localhost:8084/actuator/health   # judge-dispatcher
curl -s localhost:8080/actuator/health   # gateway
```

打开 http://localhost:8848/nacos → **服务管理 → 服务列表**,应能看到 5 个服务。

### 5.3 应用层环境变量

`application.yml` 已用 `${VAR:default}` 包好,本地全用默认值即可。要改连接地址,临时:

```bash
NACOS_ADDR=10.0.0.5:8848 mvn -pl services/user-service spring-boot:run
```

**永久**:在你 home 下放 `~/.judgemesh/local.env` 自己 `source`,**不要改仓库里的 application.yml**。

## 8. IDE 配置

### 8.1 IntelliJ IDEA(推荐 Java 端)

1. **Open** → 选仓库根目录(包含顶层 `pom.xml`)
2. 等 Maven 同步完成(右下角进度条)
3. **Settings → Build → Build Tools → Maven**:确保 JDK 是 17
4. **Settings → Plugins**:装 **Lombok** 插件
5. **Settings → Build → Compiler → Annotation Processors**:勾选 *Enable annotation processing*
6. **Settings → Editor → Code Style**:导入 `https://github.com/google/styleguide/blob/gh-pages/intellij-java-google-style.xml`(可选,统一风格)

#### 推荐运行配置

每个服务一个 Spring Boot 配置:

| Module             | Main class                                          | Active profile | Working dir              |
| ------------------ | --------------------------------------------------- | -------------- | ------------------------ |
| user-service       | `com.judgemesh.user.UserApplication`                | `local`        | `services/user-service/` |
| problem-service    | `com.judgemesh.problem.ProblemApplication`          | `local`        | `services/problem-service/` |
| submit-service     | `com.judgemesh.submit.SubmitApplication`            | `local`        | `services/submit-service/` |
| judge-dispatcher   | `com.judgemesh.dispatcher.DispatcherApplication`    | `local`        | `services/judge-dispatcher/` |
| gateway            | `com.judgemesh.gateway.GatewayApplication`          | `local`        | `services/gateway/`      |

### 8.2 GoLand(judge-worker)

- **Open** → `services/judge-worker/`
- 确认 **Settings → Go → GOROOT** 指向 1.22
- Run config:`Go Build` · package `./cmd/worker` · working dir 同上

### 8.3 VS Code(轻量替代)

装这几个扩展:`Extension Pack for Java`、`Spring Boot Extension Pack`、`Go`、`YAML`、`Docker`、`ESLint`、`Prettier`。

`.vscode/settings.json` 建议(自己加,不要 commit):

## 10. 常见问题

### 10.1 Maven 拉不到 SCA 2025.0.0.0
症状:`Could not find artifact com.alibaba.cloud:spring-cloud-alibaba-dependencies:pom:2025.0.0.0`
对策:根 `pom.xml` 改 `<spring-cloud-alibaba.version>2025.1.0.0</spring-cloud-alibaba.version>`,重跑 `mvn install`。

### 10.2 服务起不来,卡在 Nacos 注册
症状:日志反复 `Connection refused: nacos`
对策:
1. `docker compose -f infra/local/docker-compose.yml ps` 确认 nacos 是 `healthy`
2. 等 30 秒(Nacos 启动慢)
3. `curl http://localhost:8848/nacos/actuator/health`

### 10.3 Lombok 注解不生效 → 编译报红
症状:`getXxx() / setXxx()` 找不到方法。
对策:IDEA 装 Lombok 插件 + 开 annotation processing(§8.1 第 4-5 步)。

### 10.4 MySQL 连接被拒
症状:`Access denied for user 'judgemesh'@'…'`
对策:确认 `infra/local/.env` 里 `MYSQL_USER` / `MYSQL_PASSWORD` 与服务 `application.yml` 一致(默认都是 `judgemesh / judgemesh`)。

### 10.5 端口被占
```bash
lsof -i :8080 | grep LISTEN     # macOS / Linux
netstat -ano | findstr 8080     # Windows
```
找到 PID `kill` 掉,或改 application.yml 端口。

### 10.6 isolate 在 macOS 跑不通
**不是 bug**——isolate 强依赖 Linux namespaces / cgroups,macOS 内核没有。本地开发期 worker 用占位实现;Sprint 1 起 @Phoen1xCode 在 Linux 容器内测。

### 10.7 docker compose down 之后想保留数据
默认 `down` 不删数据卷;要清空写 `down -v`。注意 `-v` 是删卷,谨慎。

### 10.8 改了 services/api/ 模块后其他服务编译不通过
两步:
```bash
mvn -pl services/api -am clean install      # 先把 api 装到本地仓
mvn -pl services/<your-service> spring-boot:run
```
